第一章对象导论

John Doe 2020-04-03 20:29:27

1.面向对象编程（Object-oriented Programmer ，OOP）

2.Smalltalk 的五个基本特性 （java所基于的语言之一）

①万物皆对象

② 程序是对象的集合，他们通过发送消息来告诉彼此所要做的

③每个对象都有自己的由其他对象所构成的存储

④每个对象都拥有其类型

⑤某一个特定类型的所有对象都可以接受同样的信息

3.一个类就相当于一个数据类型，

我们创建新的类就相当于创建了一个新的数据类型，这样我们就可以不拘泥于使用现有的用来表示机器中的存储单元的数据类型

4.每个对象都提供服务（每个对象都是服务提供者）

要实现一个复杂的需求，可以使用多个类，或者交给一个类，这个类再分发给其他类处理，其他类也可再向下分发

保证每个类都专注于完成一项任务而不试图做其他事情，这样既简化了问题的复杂性，也保证了程序的高内聚性

5.被隐藏的具体实现

为什么要隐藏类中的一些成员或方法？

类创建者的目标是构建类，这种类只向客户端程序员暴露必须的部分，被隐藏的部分往往是对象内部脆弱的部分，他们很容易被客户端程序员主动或无意的修改造成类损坏，将这部分隐藏起来可以减少程序bug

public ：公开的，外部类可随意访问

private：私有的，只有本类成员能够直接使用，外部类无法访问

protected ：在同一个包中与public相同，不是同一个包与private相同

6.复用具体实现

复用类可以选择继承 ，组合，聚合

简单的复用某个类时，就可以只使用这个类的一个对象，将他作为成员对象，

静态的引入即组合 ， 动态的引入即聚合（使用某种方法，在调用该方法时才创建该成员对象）

7.继承

通俗的讲 继承就是创建一个子类（导出类，继承类）继承父类（基类，超类）的所有成员和方法

并且子类可以在父类的基础上进行拓展，覆盖，但是不可以删除，修改继承下来的成员和方法

所有的类都是继承于Object类，Object类是一切类的父类

子类在继承父类时同时复制了父类的接口，所有发送给父类对象的消息同时也可以发送给子类对象

子类与父类具有相同的类型

8.多态

在处理类型的层次结构时，经常想把一个对象不当做它所属的特定类型来对待，而是将其当做基类来对待，这使得人们可以变写出不依赖特定类型的代码。

前期绑定：编译器将产生对一个具体函数名字的调用，而运行时将这个调用解析到将要被执行的代码的绝对地址

在Oop中，程序直到运行才能够确定代码的地址，所以当消息发送到一个泛化对象 时，必须采取其他机制

后期绑定：当对象发送消息时，被调用的代码直到运行时才能确定。编译器确保调用方法的存在，并对调用参数和返回值进行类型检查（无法提供此类保证的语言被称为弱类型的），但是不知道将被执行的确切代码

java默认动态绑定 c++使用virtual 关键字实现

将导出类看做是他的基类的过程被称为向上转型

8.单根继承

所有的类都是继承于Object类

单根继承结构保证所有对象都具备某种功能

单根继承结构使垃圾回收器的实现变得容易很多

9.容器

List Map Set 堆 栈 树 等等等等

不同的容器提供了不同的接口和外部行为，不同的容器对于某些操作具有不同的效率，因此设计了不同的序列

Arraylist 访问元素快于 Linklist 增删改相反

10.参数化类型

当对象引用置于容器中时，必须向上转型为Object类型，但是在向下转型为一个具体类的时候，无法知道要转到哪个类型，因此，引出了参数化类型的解决方法

即java中的泛型 如 Arrlist< String >

使用参数化类型编译器就可以定制一个只接受和取出指定类型的容器

11.对象的创建和生命期

11.对象的创建和生命期

c++认为效率控制是最重要的议题，为追求最大执行速度，对象的存储空间和生命周期可以在编写程序时确定，这可以通过将对象置于堆栈（自动变量，限域变量）或静态存储区域内来实现。这种方式将存储空间分配和释放置于优先考虑，牺牲了灵活性。且如果不能正确的销毁对象，可能造成内存泄漏

java使用了动态方式，在堆的内存池中年动态的创建对象，因为存储空间是在运行时被动态管理的，所以需要大量的时间在堆中分配存储空间。动态方式所带来的更大灵活性正是解决一般化编程的问题要点所在。因为有java垃圾回收器的存在，因此会自动释放内存

12.异常处理

通俗的讲就是代码遇到错误会被异常处理捕捉，再执行异常处理中的办法，最终回到主程序

异常处理在面向对象语言出现之前就存在了

13.并发

即多线程，加大代码执行效率，如果操作系统支持多处理器，就可以把每个任务指派给不同的处理器

也有隐患：共享资源带来的安全隐患，这时就要增加资源锁

14.java与internet

不多赘述

面向服务器端编程 java 等

面向客户端 html 等

脚本语言 js 等

java applet 只在web浏览器运行的小程序，需要在有浏览器和java解析器的环境下使用，作为网页的一部分被下载，因为当初安装jre带宽要求大，因此没有普及

internet 与 intranet

internet面向全web，因此安全性不如 intranet

intranet大部分用于公司内部（企业内部网）可以物理的控制对公司内部服务器的访问
