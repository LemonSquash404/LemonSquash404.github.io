servlet

John Doe 2020-04-06 10:13:51

servlet是javaweb三大组件之一

三大组件分别是 filter过滤器，servlet程序 listener 监听器

servlet是一个运行在web服务器的java小程序作用是通过http协议接受客户端的请求

http://localhost:8080/Test/test1

​ http: http 协议

​ localhost ：服务器地址

​ 8080：端口号

​ Test：工程路径

​ test1：资源路径

servlet的生命周期

1.调用servlet程序的构造器方法

2.调用servlet程序的init初始化方法

（第一次访问servlet程序的时候才会调用）

3.调用servlet程序的service方法

（每次访问都会调用）

4.调用servlet程序的destroy方法

​ （web工程停止的时候调用）

ServletConfig类

ServletContext 类

ServletContext是一个接口

ServletContext是一个域对象 web创建生成，web项目停止销毁

一个web工程，tomcat只会创建出一个ServletContext对象

域对象是可以像map集合一样存储对象

保存对象

获取数据

ServletContext类的四个方法

1.获取在web.xml中配置的上下文参数<context - param>

2.获取当前工程的工程路径

3.获取工程发布到服务器上之后，资源或目录在服务器磁盘上的绝对路径‘

4.可以像map一样存放数据 销毁后数据消失

​ 1.key ：value 不同的key有不同的作用

请求体

​ 常用的请求头说明

​ Accept： 客户端可以接受的数据类型

​ Accept-Language： 客户端可以接受的语言

​ Host： 请求的服务器ip和端口号

​ User-Agent： 浏览器的信息

​ Content-Length： 请求体的长度

​ Accept-Encoding： 服务器返回的数据，支持的压缩格式（或编码格式）

  1. 响应头

常用的响应码

200 请求成功

302 表示请求重定向

404 资源不存在

500 服务器代码错误

HttpServletRequest类

tomcat会将请求内容封装到HttpServletRequest对象中，然后传给Servlet程序去用

HttpServletRequest对象由服务器创建

API

​ request.getRequestURI(); 获取请求的url路径 请求的资源地址
​ request.getRequestURL(); 获取请求的同一路径 请求的绝对路径
​ request.getRemoteHost(); 获取客户端id
​ request.getHeader(); 获取请求头
​ request.getParameter(name); 获取请求的参数值
​ request.getParameterValues(name); 获取请求的多个值
​ request.getMethod(); 获取请求方式
​ request.setAttribute(name, o); 获取域对象
​ request.getRequestDispatcher(path); 获取请求转发对象
​ request.getCookies(); 获取cookies信息
​ request.getSession(); 获取会话信息
